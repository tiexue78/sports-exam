# 项目：初二体育知识练习系统

## 背景

初二孩子准备体育与健康知识考试，老师给了出题范围和答案。需要一个在线练习系统，方便反复练习，并分享给同学使用。

## 源文件

- `3_北京市初中学业水平考试体育与健康知识试题(1).pdf` — 题目
- `选择题、判断答案(1)(1).docx` — 答案

## 核心需求

1. **题型**：选择题 + 判断题
2. **两种模式**：
   - **学习模式**：逐题练习，做完一题即可查看对错和正确答案（也可选择先不看，自己判断后再看，加深记忆）
   - **考试模式**：计时作答，完成后出成绩和对错详情。开始前可自选：测试范围（章节）、题目数量、题型（选择题/判断题/混合）
3. **进度保存**：学习和考试都支持中断后继续
4. **错题本**：自动收集错题，可集中练习
5. **历史记录**：记录每次考试情况
6. **身份识别**：输入姓名即可，重名时提示换名
7. **手机端友好**：响应式设计，主要使用场景是手机浏览器
8. **在线访问**：需部署上线，分享链接即可使用

## 技术方案

- **框架**：Next.js
- **数据存储**：Supabase（免费额度足够几十人使用）
- **部署**：Vercel（免费）
- **样式**：Tailwind CSS，移动端优先

### 数据模型（初步）

- `users`：姓名、创建时间
- `questions`：题号、题型（选择/判断）、题目内容、选项、正确答案、章节/分类
- `practice_progress`：用户ID、当前题号、模式（学习/考试）、开始时间
- `exam_records`：用户ID、考试范围、题目数量、题型筛选、考试开始时间、结束时间、总分、答对数
- `answers`：用户ID、题目ID、用户答案、是否正确、作答时间
- `wrong_questions`：用户ID、题目ID、错误次数、最近错误时间

## 用户确认的需求细节

1. 题目数据导入 Supabase（同意原建议）
2. 登录时确认姓名，默认显示上一次登录的姓名
3. 考试计时仅展示，不限制时间，暂停后从暂停处继续
4. 考试题目随机抽取
5. 每个步骤执行完后自检
